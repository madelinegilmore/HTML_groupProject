User Profile Page

HTML Structure

The User Profile page was built using semantic HTML elements such as <header>, <main>, and <section> to ensure a clear and accessible structure. The layout follows a modular design where each profile component is separated into logical blocks.

CSS Techniques

Flexbox was used to align profile components and ensure responsive layout behaviour across different screen sizes. Hover effects and spacing were applied to improve usability and visual feedback.

JavaScript Implementation

JavaScript was used to create an interactive multi-step profile system. User input was collected using prompt() and stored in variables. DOM manipulation was used to dynamically update the page without requiring reloads.

Conditional logic allowed users to skip optional questions, while a progress tracker was implemented to show completion status.

Design Rationale

A step-by-step interaction model was chosen to improve usability and reduce cognitive load for users. The decision to avoid traditional forms was intentional to demonstrate dynamic JavaScript control.

Sitemap

HTML Structure

The sitemap was implemented using SVG elements such as <rect>, <line>, and <text> to visually represent the website structure and hierarchy.

CSS Techniques

CSS was used to enhance interactivity through hover states, transitions, and focus styling for keyboard navigation. SVG elements were styled using stroke and fill properties for clarity.

Design Rationale

An SVG-based approach was chosen because it allows scalable, resolution-independent diagrams, which is ideal for representing structured navigation systems.

Content Page

HTML Structure

The Content Page uses semantic structure with multiple <section> elements and internal anchor links. This improves navigation and allows users to jump between content sections efficiently.

CSS Techniques

CSS was used for layout spacing, typography control, and image styling. A fixed-position "Back to Top" button was implemented to improve long-page navigation.

Design Rationale

The internal navigation system was introduced to improve user experience when reading long-form content. Semantic HTML ensures screen readers can interpret the page structure correctly.

Challenges and Lessons Learned

One major challenge was implementing dynamic JavaScript interactions using prompt-based input while maintaining a smooth user experience. Managing state across multiple steps required careful structuring of variables and functions.

Another difficulty was designing the SVG sitemap layout, as minor coordinate changes significantly affected positioning. This was resolved through iterative testing and adjustment.

Overall, this project improved my understanding of DOM manipulation, SVG graphics, and modular web design principles.

Accessibility Considerations

Accessibility was considered throughout development by using semantic HTML elements, meaningful headings, and alternative text for images.

Keyboard navigation was supported through focusable SVG elements and visible focus indicators. Colour contrast and spacing were designed to improve readability for users with visual impairments.

JANET Compliance

All pages comply with JANET regulations by ensuring that content is appropriate, ethical, and academically valid. No offensive, illegal, or inappropriate material is included.

External sources were used responsibly and referenced where applicable. The project follows academic integrity guidelines by ensuring originality of code and proper attribution of learning resources.
