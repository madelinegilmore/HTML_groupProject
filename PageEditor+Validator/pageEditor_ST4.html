Sitemap Page

Technical Description

The sitemap was created using SVG elements such as rect, line and text to visually represent the website structure. Each page is displayed as a node connected by lines to show hierarchy.

Interactive features were added by wrapping nodes in anchor tags, allowing users to click and navigate. CSS hover effects and focus styles were used to improve usability.

Accessibility

Each node includes text labels and title elements for screen readers. Keyboard navigation is supported using tabindex, and focus styles are clearly visible.

Content Page

Technical Description

The content page was structured using semantic HTML elements including section and headings to divide content into clear parts. Internal navigation links were added at the top of the page using anchor links to allow users to jump between sections.

CSS was used to style images, spacing and layout. A fixed "Go to Top" button was implemented using position: fixed to ensure it stays visible while scrolling.

Accessibility

All images include alt text and headings are used in a logical order to improve readability. Internal navigation helps users quickly move through long content.

Challenges and Lessons Learned

One challenge was implementing JavaScript prompts while still making the page feel interactive. It was also difficult to manage updating the profile dynamically without reloading the page.

For the sitemap, positioning SVG elements correctly took time as small changes in values affected the layout. I solved this by testing different coordinates and adjusting step by step.

Overall, I improved my understanding of DOM manipulation, SVG usage and page structure.
