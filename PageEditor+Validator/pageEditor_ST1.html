Home page

Technical Description

I made a simple home page that is easy to navigate. I used the standard HTML structure we learned in class (header, main, and footer) because they naturally split the page into clear sections, and that made it easier to know where everything belongs. The navigation uses links, and an active class shows users which page they’re currently on. For the content, I used cards because they make each topic separate and easier to scan. Wrapping each card in a link made it user-friendly so people can click anywhere on the card instead of aiming for a tiny button. For the styling, I used CSS Grid for the card layout for a two-column structure (display: grid, grid-template-columns, and gap). I kept the images consistent using fixed width/height with object-fit: cover, to cut off any excess and keep them uniform.

Accessibility

For accessibility on this page, I first used proper semantic structure (header, nav, main, footer) so screen readers can understand the page layout clearly. I kept headings in a logical order so content is easier to follow. I also made sure images had alt text that describes what the image is about so users who can’t see the images still get the context. I also fixed some HTML issues that affect accessibility tools, like invalid tag nesting, because this makes for better compatibility with screen readers and validators. In navigation and interactive parts, I used clear link text and visible hover/active states so users can tell what is clickable and where they are in the site.

Gallery Page

Technical Description

For the gallery page, I also built it around a simple semantic structure with header, nav, main, and footer so it stayed consistent with the rest of the site and easy to follow. Inside the main section, I used a horizontal card layout (.scroll-container and .image-card) so users can browse ecosystem topics visually. Each card stores its description and title in data-title and data-desc. This made the JavaScript clean because I could reuse the same logic for every card instead of hardcoding each one. For CSS, I used positioning and overlays to show quick info on hover, then I added a full-screen modal for deeper interaction when a card is clicked. The modal uses a split layout (image and text side by side). JavaScript handles the interactivity by creating overlays, opening/closing the modal, switching card content based on the clicked image, supporting escape-key close, and adding the light/dark mode toggle for the card text panels.

Accessibility

For accessibility, I kept semantic page landmarks so assistive tech can navigate the page more predictably, and I made image alt text topic-specific so the visual can be interpreted by screen readers. I also fixed validation issues like improper ARIA usage (for example, adding an appropriate role when using aria-label) and avoided invalid tag nesting patterns that can confuse accessibility tools. For interactive elements, I added clear labels (like the modal close button’s aria-label) and keyboard support (Escape to close modal), which helps users who don’t rely on a mouse. The light/dark mode toggle makes the page more accessible for users with different visual needs.

Content Page

Technical Description

For species.html, I kept the same overall page structure as the rest of the project (header, nav, main, footer) to keep everything consistent. Inside the main section, I split content into separate cards for each species, which made the page easier to scan and gave each animal its own focused section. I used anchor links at the top (#african-elephant, #sea-turtle, etc.) so users can jump straight to the section they want instead of scrolling through everything. For the CSS, I made sure that the images are centered and sized consistently with main img, and spacing is handled with card containers and basic margin/line breaks.

Accessibility

I added alt text to all species images so the visuals still have context for screen reader users. I also fixed HTML tag issues that were causing validation problems, like removing invalid nesting (for example, headings inside paragraph tags) and correcting list structure so all list content sits inside proper li elements. I also fixed the button/a nesting error at the “top of page” control, which improves both validity and accessibility behavior. Challenges and Lessons Learned

A challenge I faced was working on the gallery page. I had to figure out how to make the cards scroll horizontally and how to make the modal work. W3 schools was a great resource for this. I combined the gallery scroll and modal functionality from the template to my own design ideas. I also had to figure out how to make the light/dark mode toggle work. I ended up using a checkbox input and JavaScript to toggle the class on the body element.
